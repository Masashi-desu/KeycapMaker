# SCAD 設計方針メモ

## プレビュー用と本番出力用の分離

- プレビューは操作感を優先し、軽量化を許容する
- 本番出力は寸法と形状の正確性を優先する
- 同じパラメータを受け取っても、preview と export で別エントリを持てる設計が望ましい
- プレビュー専用の簡略 legend や低コスト形状を許容する

## stem / legend / profile / preset の設計観点

### stem

- 取り付け方式ごとの差分が大きいため、モジュールとして分離しやすい
- ボディ基準座標との関係を固定し、他要素から独立して差し替えやすくする

### legend

- `text()` 依存、フォント依存、立体化方式の影響を受けやすい
- body と別体積で扱う前提があるため、配置座標と厚みの責務を明示する

### profile

- 高さ、傾き、上面形状など、全体シルエットへ影響する
- ベース形状の派生として扱うか、プリセット主導で吸収するかを早めに決める

### preset

- UI 初期値と SCAD パラメータの橋渡しになる
- キー種別ごとの差分をまとめる場として使う

## 将来の拡張で必要になる可能性が高い項目

- キーサイズごとの寸法体系
- legend の複数行、複数位置、アイコン対応
- フォント切り替えと同梱ルール
- profile 別のトップ面形状
- ステム方式の追加
- プリンタやスライサー固有の export プリセット

## 印字を別体積で扱う場合の考え方

- body と legend の原点関係を一定に保つ
- legend 側の厚みと埋まり量を明示的に制御する
- 交差させるのか、接地させるのかを export 仕様として決める
- スライサーで部品として扱うことを想定し、単なる色属性に頼らない

## パラメータ設計時の責務分離

- UI パラメータ: 入力しやすさと検証しやすさを優先する
- SCAD パラメータ: 幾何生成に必要な値へ正規化したものを使う
- export パラメータ: 形式別の制約やボディ分離方針を反映する
- preview パラメータ: 軽量化のための近似や省略を許容する

## 推奨構成

- `scad/base/`: export エントリと全体形状
- `scad/modules/`: 差し替え可能な小さな構成要素
- `scad/presets/`: プロファイルとキー種別の初期値
- `scad/samples/`: 回帰確認と PoC 用サンプル
