# Bambu Studio 読み込み確認チェックリスト

このチェックリストは [../roadmap/implementation-plan.md](../roadmap/implementation-plan.md) の Task 05 を実施するための固定手順です。人間はこの文書の順で確認し、結果だけを Codex に返してください。

## 対象ファイル

- `minimum-poc.stl`
- `minimum-poc.3mf`
- 将来追加される `body.stl`, `legend.stl` または多ボディ 3MF

## 確認項目

1. 読み込み成功可否
2. 寸法の違和感有無
3. 向きの違和感有無
4. メッシュ破綻の有無
5. 複数ボディがある場合:
   body / legend が別オブジェクトとして扱われるか
6. 3MF の場合:
   パーツ構造や色の解釈が崩れないか

## 返答テンプレート

以下の形式で Codex に返すことを推奨します。

```text
対象: minimum-poc.stl
読み込み: 成功 / 失敗
寸法: 問題なし / 問題あり
向き: 問題なし / 問題あり
メッシュ: 問題なし / 問題あり
備考: 任意
```

```text
対象: minimum-poc.3mf
読み込み: 成功 / 失敗
寸法: 問題なし / 問題あり
向き: 問題なし / 問題あり
メッシュ: 問題なし / 問題あり
複数ボディ: 該当なし / 維持 / 崩れる
色・パーツ解釈: 該当なし / 問題なし / 問題あり
備考: 任意
```

## 判断ルール

- STL が安定して読める場合は、最低保証形式として維持する
- 3MF に不安定要素がある場合は、色付き 3MF を本番前提にしない
- 色またはパーツ解釈に不安がある場合は、Task 06 で body / legend 別体積方式を正式候補とする
