- 次の対応: 追加パラメータを増やす際も、wrapper 生成と `user_*` 定義を同時に更新して回帰確認する

### 2026-04-18 - Task 08 legend text 入力とフォント同梱

- 対象タスク: Task 08
- 結論: legend を固定ブロックから `text()` ベースへ切り替え、UI から任意文字列を入力できるようにした。実行時には `public/fonts/MPLUS1p-Regular.ttf` を OpenSCAD runtime へ渡す

### 2026-04-18 - Task 08 複数書体対応と UI 文言の一般向け化

- 対象タスク: Task 08
- 結論: 印字入力を印字カードへ統合し、複数文字入力と書体選択を追加した。書体は `M PLUS 1p`、`M PLUS Rounded 1c`、`DotGothic16` を同梱し、各項目名と説明はエンドユーザ向けの表現へ寄せた
- 理由: 内部実装寄りの用語を避けつつ、配信に同梱できるライセンスの書体だけを安全に選べるようにするため
- 次の対応: 人間が実ブラウザで文言の分かりやすさと、各書体の見え方を確認する

### 2026-04-18 - Task 08 印字の装飾オプション追加

- 対象タスク: Task 08
- 結論: 書体表示は正式名称へ統一し、印字には太字、イタリック風、ななめ、下線を選べるようにした。装飾は追加フォント依存ではなく、`text()` の 2D 形状へ `offset()`、`multmatrix()`、補助矩形を適用して表現する
- 理由: 同梱フォントごとに利用可能なスタイル差が大きいため、書体によらず同じ選択肢を安定提供できる方法を優先した
- 次の対応: 人間が実ブラウザで各装飾の視認性を確認し、必要なら傾き量や太字量を補正する

### 2026-04-18 - Task 08 印字 UI の表示制御と高さ 0 の意味づけ

- 対象タスク: Task 08
- 結論: 印字の詳細項目は `印字を入れる` をオンにしたときだけ表示し、既定文字は `A` に変更した。`文字の高さ` は「表面からの出っ張り量」として扱い、既定値 `0` は表面と同じ高さで見える状態を指す
- 理由: 印字を使わない利用者に不要な入力欄を見せず、`0` を無効値ではなく flush な初期状態として分かりやすく扱うため
- 次の対応: 人間が実ブラウザで印字オフ時の見え方と、`高さ 0` の見た目・視認性を確認する

### 2026-04-18 - Task 08 印字露出面のカーブ追従

- 対象タスク: Task 08
- 結論: legend は flat な押し出しをそのまま置くのではなく、dish と同じ円筒カーブ帯で切り抜く構成へ変更した。これにより、`文字の高さ = 0` でもトップ面のカーブに沿って文字が全面で露出する
- 理由: 文字本体を中心高さ基準の平面で止める方式だと、dish の高い側で body に埋もれて文字の一部が見えなくなるため
- 次の対応: 人間が実ブラウザで複数文字や位置移動時にも露出欠けが起きないかを確認する
